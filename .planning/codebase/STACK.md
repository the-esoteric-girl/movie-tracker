# Technology Stack

**Analysis Date:** 2025-03-09

## Languages

**Primary:**
- JavaScript (ES2020+) - All source code in `src/`

**Secondary:**
- JSX - React component syntax in `src/components/` and `src/pages/`

## Runtime

**Environment:**
- Node.js - No version constraint specified in package.json

**Package Manager:**
- npm - Lockfile present at `package-lock.json`

## Frameworks

**Core:**
- React 19.2.0 - UI library, imported in `src/main.jsx`, `src/App.jsx`, all components
- React Router DOM 7.13.1 - Client-side routing, configured in `src/main.jsx` with `BrowserRouter`, routes in `src/App.jsx`

**Build:**
- Vite 7.3.1 - Build tool and dev server, configured in `vite.config.js`

**UI Plugin:**
- @vitejs/plugin-react 5.1.1 - Vite plugin for React Fast Refresh, used in `vite.config.js`

## Key Dependencies

**Critical:**
- react@19.2.0 - Core UI framework for component rendering
- react-dom@19.2.0 - React DOM rendering, mounted in `src/main.jsx` via `createRoot()`
- react-router-dom@7.13.1 - Routing engine handling `/`, `/saved`, `/seen`, `/you`, `/movie/:id` routes defined in `src/App.jsx`

## Development Dependencies

**Linting & Code Quality:**
- eslint@9.39.1 - JavaScript linter, config at `eslint.config.js`
- @eslint/js@9.39.1 - ESLint recommended rules
- eslint-plugin-react-hooks@7.0.1 - React hooks linting rules in `eslint.config.js`
- eslint-plugin-react-refresh@0.4.24 - React Refresh plugin for Vite

**Type Support:**
- @types/react@19.2.7 - TypeScript definitions for React (JSX support, not strict TS)
- @types/react-dom@19.2.3 - TypeScript definitions for React DOM

**Utilities:**
- globals@16.5.0 - ESLint browser globals configuration in `eslint.config.js`

## Configuration

**Environment:**
- `.env` file present at project root
- Runtime API key loaded via `import.meta.env.VITE_TMDB_API_KEY` in `src/pages/FindPage.jsx` and `src/pages/MovieDetailPage.jsx`
- Vite env variable prefix: `VITE_` (automatically exposed to client-side code)

**Build:**
- `vite.config.js` - Minimal config, enables React plugin only
- `eslint.config.js` - Flat config format with rules:
  - Targets `**/*.{js,jsx}` files
  - Browser globals enabled
  - `ecmaVersion: 2020`
  - JSX support enabled
  - Extends: `js.configs.recommended`, `reactHooks.configs.flat.recommended`, `reactRefresh.configs.vite`
  - Custom rule: `no-unused-vars` ignores uppercase/underscore patterns (e.g., `_unused`, `CONSTANT`)

**Entry Points:**
- HTML: `index.html` - Single page entry point with `<div id="root">` mount point
- JavaScript: `src/main.jsx` - React root initialization with StrictMode and BrowserRouter

## CSS & Styling

**Approach:**
- CSS Modules for component scoping: `MovieDetailPage.module.css`, `SearchBar.module.css`, etc.
- Global CSS: `src/index.css` (design tokens, custom properties), `src/App.css` (global layout classes)
- No CSS-in-JS library

**Design System:**
- CSS custom properties for colors, spacing, typography, shadows
- Global layout classes in `src/App.css` (`.app`, `.heading`, `.movie-grid`, `.empty-state`)

## Platform Requirements

**Development:**
- Node.js runtime (no version specified, npm required)
- Text editor with JSX/React support recommended

**Production:**
