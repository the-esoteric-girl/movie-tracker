Design decisions

  • Search lives inside each tab as a behaviour, not a destination screen — avoids dead-end navigation and keeps users oriented
  • Movie Detail is a global screen reachable from three tabs — represented separately to avoid duplication in the map
  • Log Movie Modal is the only overlay child of Movie Detail — all other actions are stateful buttons on the same screen
  • Onboarding is a first-launch-only flow that exits directly into the Find tab
  • "You" tab is intentionally minimal in V1 — architected to expand into a full profile in V2

V2 considerations

  • Nav "You" tab grows into full Profile — build the tab to support expanded content
  • Auth V2 adds login/signup screens — onboarding flow will branch here
  • Social Friend activity, watchlist overlap, and likes/comments add new screens under "You" and Movie Detail
  • Data Use TMDB movie IDs as canonical reference throughout — required for social layer
